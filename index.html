Gerações dos computadores

Os computadores de primeira geração usavam válvulas eram lentos e enormes.

A segunda geração substituiu as válvulas por transistores. Isso tornou os computadores menores e de custo mais baixo.

A terceira geração de computadores foi construída com circuitos integrados, proporcionando maior compactação, redução dos custos.

A quarta geração aperfeiçoa os computadores, otimiza as máquina para os problemas do usuário.

A quinta geração seria a Inteligência Artificial (IA), compuatdores quânticos, moedas digitais (bitcon) e realidade aumentada (ou a virtual).

Como surgiu a internet

Antigamente a comunicação era realizada por Telégrafos, que usavam código morse para se comunicar. Os fios usados nesse tipo de comunicação atravessavam o oceano, eram lentos, quebravam facilmente e a mensagem nem sempre vinha sem ser corrompida.

Na década de 50, por causa da guerra fria, os EUA criaram a DARPA para criar idéias que ajudassem na guerra fria.

Em algum momento, Joseph Carl Robnett licklider teorizou sobre um rede onde todos poderiam se comunicar através dela. E sobre está teoria, foi criado o Arpanet pela DARPA.

A arpanet foi um sistema de redes conectadas pelo país. Com seu advento também veio a criação de protocólos de comunicação, como o TCP/IP.

Outra figura importante foi o Tim Berners-Lee que uniu um hipertexto com TCP/IP e criou o World Wide Web (www) para criar sites onde todos podessem acessar.

O que são Clients?

Conceito

Cliente é a maquina (computador) usado pelo usuário final.

A partir da maquina do cliente que é feito os pedidos (requisição) de algo que se encontra em algum servidor. Essa requisição passa pela internet, encontra o servidor que hospede a informação que você deseja, e retorna uma cópia para o seu computador.

Hoje em dia, no lado do cliente, existe vários dispositivos para se conectar em sites e aplicativos, porém, esses dispositivos tem tamanho de telas diferentes e as vezes até em modo retrato. Por isso foi criado o conceito de Design Responsivo (ou responsividade), que se espera criar um site ou aplicativo que seja consistente com qualquer tamanho de tela.

Boas práticas com o lado do cliente é testar se ele pega em navegadores diferentes e em sistemas operacionais diferentes (ou a própria versão de um sistema em uma maquina desatualizada).

Navegadores

São programas criados por empresas, utilizados para abrir/executar arquivos.

Seguem padrões W3C, porém, sempre tem uma diferença ou outra de inter´retação.

São chamados de "browsers". Teve seu inicio com o MOSAIC, passou para o Netscape, e hoje tem uma variedade de navegadores disponíveis.

Aplicações Web

Aplicações Web são soluções criadas que a internet como meio de comunicação Client x Server não sendo necessário a sua instalação.

Não confundir com apliacativos para celular ou desktop, que necessitam de intalação e podem ser usados em modo offline.

Dispositivos móveis

Antigamente programadores só precisavam pensar no desktop. Mas hoje em dia, ha uma gama muito grande de dispositivos, principalmente moveis, de telas de tamanhos distintos. Então, sempre que for criar uma aplicação web, deve se levar em conta que a aplicação pode ser aberta em qualquer dispositivo com internet.

Sobre Servidores

Conceito

Os servidores ficam do lado oposto do clients. É nos servidores que ficam armazenado as informações requisitadas pelo cliente, ou ficam salvas a localização de tais informações.

LAN (Local Area Network) é quando não precisa de um meio para o cliente se comunicar com o servidor, pois o servidor já se encontra na empresa (talvez armazenando dados confidencias).

Uma das vantagens do servidor é a sua capacidade de atender diversos Clients ao mesmo tempo, pesquisando e retornando suas requisições.

Existem vários tipos de Servers: Arquivos, segurança (Firewall), Streaming, e-mail e web.

Sobre hardwares e softwares em servidores

Softwares: Eles vem com sistemas operacionais, monitoramento e servidores web.

hardwares: Contam com armazenamento (disco rígido e SSDs), memórias e processadores.

Existe sistemas operacionais feitos exclusivamente para servidores, como tambem aplicações para gerenciar servidores web (como o APACHE).

Ou seja, dentro de cada servidor existe um sistema operacional que lida com o hardwares, e um servidor web para lhe dar com as requisições feitas pelo client.

Tipos de servidores

Proxy é um servidor que regula em quais sites não pode ser acessado, protegendo assim os computadores da empresa contra virus de sites sem segurança.

Firewall projete os computadores da empresa e a rede de ser invadida por pessoas que não tem permissão para acessar certos dados.

Web Server é onde fica armazenado as aplicações Web, só precisa de internet para acessar esses arquivos.

E-mail Server é onde fica armazenado os e-mail enviados e recebidos.

Database Server é onde fica armazanado o banco de dados de uma aplicação (web ou não).

Intranet usa servidores sem precisa da internet como meio.

DNS (Domain Name Service) é um servidor que salva a conversão entre o nome de dominio de um site e seu IP, para localizar onde o site foi armazenado.

Hospedagem de sites

Para colocar seu site no ar, é preciso de um local para ele ficar salvo, logado a internet e 24h disponivel para o publico. Com isso em mente, devemos pagar para uma empresa de hospedagem manter nosso site salvo.

Antes de hospedar um site, é bom comprar um nome para o dominio que se deseja para o seu site, pois se já existir, você não poderá usar o mesmo nome.

Usando algumas ferramentas, como o Filezilla e a conexão FTP, você envia os arquivos do site que criou para o servidor que você contratou. E assim você tem seu site disponível para todos que quiserem acessar seu site.

O que é linguagem de programação

conceito

Linguagem de Programação é uma linguagem escrita e formal que especifica um conjunto de instruções e regras usadas para gerar programas (software).

Linguagem Server-side

Linguagens Server-side lidam com as requisições feitas pelo Client, e elas são executadas no servidor.

Das linguagens existentes, as mais famosas são:

  1. PHP
  2. C# (CSharp)
  3. NodeJS
  4. Java
  5. Ruby

Linguagem Client-side

São as linguagens executados do lado do client (no seu navegador ou desktop).

A unica linguagem de programação usada do lado do client é o JavaScript.

Curiosidade: O JavaScript também roda do lado do servidor, usando o motor V8 do google para ser interpretado. O nome do JS do lado do servidor é NodeJS.

HTML é uma linguagem de programação?

Não. HTML é uma linguagem de marcação, que serve para nomear cada item de uma página.

Aprendendo HTML na prática

Ferramentas utilizadas

Tudo que precisa para rodar HTML é um bloco de notas e um navegador. Mas é bom usar uma IDE, pois, ela trás diversas vantagens e produtividade na hora de codar.

Usando o Inspetor de Elementos

O inspetor permite ver a estrutura HTML inteira de uma página. Como tambem o estilo de cada item da página. Muito bom para fazer teste e modificações.

Estrutura básica do HTML

Um arquivo HTML tem que ser salvo com a extensão ".html".

Primeiro colocamos o primeiro item, a tag chamada "html", dentro dela vai ter as tag "head" e logo abaixo a tag "body".

A tag "head" vai por alguns meta dados, como nome de autor da página, palavras chaves da página, links de estilo, titulo da página, padrão usado para leitura de escrita, etc.

A tag "body" é onde vai ficar tudo que vai aparecer visualmente para o usuario, vai ser onde estará os textos, imagens, botoes e ademais itens da página.

Falando sobre Tags

Uma tag marcar uma função para os dados que estiverem dentro dessa tag.

Toda tag tem sua abertura, e seu fechamento. No meio desses dois tem o conteudo da tag, que normalmente são textos.

Toda tag começa com um < após isso vem o nome da tag (e caso precise atributos que modificam a tag, mais seus respectivos valores modificados), e o fechamento da tag de abertura com o simbolo >. Então se coloca um conteúdo que pode ser texto ou outras tag, e se fecha a tag usando o simbolo < uma barra / e logo em seguida o nome da tag, após isso se fecha a tag de fechamento com o simbolo >.

Atributos básicos das Tags

Após criar uma tag, se desejar modificar algo em específico nela, pode se usar atributos da tag. Esses atributos são escritos na tag de abertura, logo após o nome da tag. você digita o atributo que deseja mudar, como tamanho (height) ou titulo (title), e com o sinal de igual e aspas (=""), você coloca o novo valor do atributo entre as aspas.

Falando sobre textos

Existe muitas formas de escrever textos no navegador, os mais comuns são:

Hn onde n vai de 1 a 6, mostrando que 1 é o titulo mais importante e por isso o mais grande e destacado. Enquanto que quando maior o número do titulo, menor é sua importancia e menor é seu tamanho.

p a tag de paragrafo, como já diz, serve para escrever paragrafos.

Outras tags para texto são: i, u, strong que servem para estilizar o texto, deixando ele em negrito ou italico.

Lista ordenada e não ordenada

Outra tag importante é as tag que criam uma lista, existe duas importantes, a tag "ul" e a tag "ol". A tag "ul" criar uma lista não ordenada, ou seja, mostra pontos na frente de cada item sem determinar uma ordem. Já a tag "ol" cria lista ordenadas, onde cada item tem um número, ou uma representação de ordem em seus itens. As duas tags precisam da tag "li" dentro delas para criar um item da lista.

Links

Os links são muito importante na Web, pois permitem que se conecte várias páginas de uma mesmo site ou de outros. Para se usar um link, se usa a tag "a", ela vem com um atributo chamado "href" que vai ter como valor o endereço http da onde o link aponta. Ha outros atributos que permitem a tag "a" abrir o link na mesma página ou em outras.
